# 🔍 AI 代码审查助手

> **本项目由阿里云ESA提供加速、计算和保护**

## 📖 项目简介

**AI 代码审查助手** 是一个基于阿里云 AI 模型的智能代码审查工具。通过 ESA 边缘函数代理调用阿里云通义千问等 AI 模型，实现代码的智能审查、问题检测和改进建议，帮助开发者快速发现代码问题，提升代码质量。

### 核心功能

- 💻 **多语言支持**：支持 JavaScript、Python、Java、TypeScript、C++ 等多种编程语言
- 🔍 **智能审查**：自动检测代码中的错误、警告和建议
- 📊 **评分系统**：0-100 分评分，优秀/良好/需改进/不合格四个等级
- 💡 **改进建议**：提供针对性的代码改进建议
- ⚡ **边缘加速**：利用 ESA 边缘节点，实现低延迟审查

## 🛠️ 技术栈

- **前端框架**：Vue 3 + TypeScript + Vite
- **UI 风格**：深色主题，技术化设计
- **边缘计算**：阿里云 ESA 边缘函数
- **AI 模型**：阿里云通义千问（Qwen）
- **部署平台**：阿里云 ESA Pages

## 🚀 快速开始

### 环境要求

- Node.js >= 18.0.0
- npm >= 8.0.0

### 安装步骤

1. **克隆项目**
```bash
git clone <your-repo-url>
cd ai-code-reviewer
```

2. **安装依赖**
```bash
npm install
```

3. **配置环境变量**

创建 `.env` 文件：
```env
# 边缘函数 URL（如果未配置，将使用模拟模式）
VITE_EDGE_FUNCTION_URL=/api/reviewer

# 开发模式：设置为 true 时始终使用模拟数据
VITE_USE_MOCK=false
```

**注意**：如果边缘函数未部署，系统会自动使用模拟模式，方便本地开发和测试。

4. **启动开发服务器**
```bash
npm run dev
```

5. **构建生产版本**
```bash
npm run build
```

## 📦 部署到阿里云 ESA Pages

### 部署步骤

1. **准备代码仓库**
   - 将代码推送到 GitHub 公开仓库

2. **部署边缘函数**
   - 在 ESA 控制台创建边缘函数
   - 将 `edge-function/reviewer.js` 部署到边缘节点
   - 配置环境变量：
     - `ALIYUN_ACCESS_KEY_ID`
     - `ALIYUN_ACCESS_KEY_SECRET`
   - 获取边缘函数的访问 URL

3. **在 ESA Pages 创建站点**
   - 登录 [阿里云 ESA 控制台](https://esa.console.aliyun.com/)
   - 进入「Pages」服务
   - 点击「新建站点」，选择「GitHub 仓库」
   - 连接你的 GitHub 账号并选择仓库

4. **配置构建设置**
   - **构建命令**：`npm install && npm run build`
   - **输出目录**：`dist`
   - **Node 版本**：选择 18.x 或更高

5. **配置环境变量**
   - 在 ESA Pages 站点设置中添加：
     - `VITE_EDGE_FUNCTION_URL`（边缘函数的访问 URL）

6. **部署**
   - 点击「部署」按钮
   - 等待构建完成
   - 获取部署后的访问 URL

## 🎯 使用说明

1. **配置 API Key**（可选）：
   - 点击右上角「配置」按钮
   - 输入阿里云 AccessKey ID 和 Secret
   - 配置保存在浏览器本地，不会上传到服务器

2. **输入代码**：
   - 在左侧代码编辑器中输入或粘贴代码
   - 选择代码语言（JavaScript、Python、Java 等）

3. **开始审查**：
   - 点击「开始审查」按钮
   - 等待 AI 处理（约 1-2 秒）
   - 查看审查结果

4. **查看结果**：
   - **评分**：0-100 分评分和等级
   - **问题列表**：错误、警告和建议
   - **改进建议**：针对性的代码改进建议
